# ECHO+

> **ECHO+** — 一款 Android 平台的多功能串口/网络调试助手，专为嵌入式开发调试场景设计。  

**技术栈**: Android + Kotlin (AGP 9.2.1, Kotlin 2.1.0)  
**最低 SDK**: 26 (Android 8.0) | **目标 SDK**: 36 (Android 14)

---

## 📋 功能总览

### ✅ 已实现

#### 通信协议

| 功能 | 说明 |
|------|------|
| **UDP 收发** | 支持远程/本地端口配置，异步接收线程，状态回调 |
| **TCP 客户端收发** | 支持 IP/端口/握手数据/超时配置，异步连接，断线重连回调 |
| **TCP 服务端收发** | 支持监听端口配置，多客户端接入，连接列表管理 |

> ⚠️ **串口 (USB Serial)** 和 **蓝牙** 暂未实现

#### 数据协议引擎

| 引擎 | 说明 |
|------|------|
| **FireWater** | CSV 风格字符串协议，支持采样数据与图像数据包接收 |
| **JustFloat** | 小端浮点数组字节流协议，支持采样数据与图像数据包接收，强制 Hex 模式 |
| **RawData** | 纯透传模式，原始字节直接转 hex 输出 |

- 详细功能参考[VOFA数据引擎文档](https://www.vofa.plus/docs/learning/dataengines/introduce)

#### 输出显示

| 功能 | 说明 |
|------|------|
| Hex/Abc 切换 | 实时切换十六进制 / 文本显示 |
| 时间戳显示 | 开关控制，格式 `[HH:mm:ss.SSS]` |
| Rx/Tx 开关 | 独立控制接收/发送数据显示，颜色区分 |
| 采样数据包过滤 | 独立开关控制合规采样帧的显示 |
| 图像数据包过滤 | 独立开关控制图像数据包的显示 |
| 编码选择 | UTF-8 / GBK 切换 |
| 字体缩放 | A+ / A- 步进调整 |
| 控制字符转义 | 文本模式下 `\n`, `\r`, `\t` 等转义为可见序列 |
| 数据导出 | 导出到系统 `Download/ECHO-*.log` |
| 清除输出 | 一键清空显示缓冲区 |
| 行尾追加选择 | 发送时可选追加 `\n` `\r` `\n\r` `\r\n` |
| 发送缓冲区持久化 | 编辑框内容实时保存，重启恢复 |

#### 图像显示

| 功能 | 说明 |
|------|------|
| **GPU 硬件加速渲染** | OpenGL ES 2.0 灰度图渲染 |
| **可拖拽/缩放叠加窗口** | 悬浮于主界面的独立图像窗口，支持触摸拖拽调整位置和尺寸 |
| **图像帧还原** | 开关「图像」复选框时保留并恢复上一帧图像数据 |
| **保存图像为 PNG** | 导出到 `~/Download/ECHO+/` 目录 |
| **图像信息面板** | 控件页显示当前帧的宽度、高度、格式 |
| **多协议支持** | FireWater / JustFloat 引擎均支持图像数据包解析与渲染 |
| **位置/尺寸冷保存** | 图像窗口的位置和尺寸自动持久化，下次启动自动恢复 |

#### 参数缓存与数据监测

- **ParamBuffer** — FireWater/JustFloat 合规帧自动锁定参数数量(≤32)，16 色调色板，实时 FPS 估算
- **ImageBuffer** — 最新完整一帧图像的帧头 + hex 数据，10MB 上限
- **E 按钮** — 清除参数缓存并重新锁定
- Rx 关闭时 ParamBuffer/ImageBuffer 仍正常接收数据

#### UI 与交互

- 侧边菜单栏 + 双抽屉导航（协议/设置/命令/控件页面）
- 协议页面：数据引擎选择、接口选择、各协议参数配置
- 设置页面：界面尺寸切换、TCP 超时、导出数据、恢复默认配置
- **界面尺寸适配**：设置页切换「窄（适合手机）/ 宽（适合平板）」模式，联动调整
- 面板分割拖拽（垂直/水平，60fps 防抖，像素精确最小尺寸限幅）
- 沉浸模式、连接锁定引擎、双击退出、屏幕常亮、启动淡出动画
- 配置持久化（JSON 序列化、自动保存、恢复默认、启动恢复）
- 应用基础设施（AppLogger 日志文件轮转、ConnectionLogger 连接日志、全局崩溃异常捕获）
